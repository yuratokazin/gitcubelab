# Конструкторы статических сайтов

Конструкторы и генераторы статических сайтов (SSG) создают быстрые, безопасные HTML-страницы из текстовых файлов (Markdown) и шаблонов. Топ-инструменты: **Hugo** (очень быстрый), **Jekyll** (популярен с GitHub Pages), **Eleventy** (гибкий JS), **Astro** (современный), **Pelican** (Python). Идеальны для блогов, документации и лендингов, обеспечивая высокую скорость загрузки. 

Лучшие генераторы статических сайтов (SSG)

- **Hugo**: Написан на Go, известен невероятной скоростью сборки. Отлично подходит для больших сайтов.
- **Jekyll**: Самый популярный, стандарт для GitHub Pages, написан на Ruby. Идеален для блогов.
- **Eleventy (11ty)**: Простой и гибкий генератор на Node.js, не привязан к конкретному JS-фреймворку.
- **Astro**: Современный инструмент, ориентированный на скорость и  производительность, позволяет использовать компоненты Vue, React,  Svelte.
- **Pelican**: Использует Python, подходит для создания контентных сайтов. 

Популярные визуальные конструкторы (с выгрузкой статики)

- **Mobirise AI**: Десктопный конструктор с ИИ, создающий готовый HTML/CSS/JS код, который можно разместить где угодно.
- **8B AI Builder**: Простой онлайн-конструктор статических сайтов с ИИ. 

Преимущества статических сайтов

1. **Скорость**: Мгновенная загрузка, так как сервер отдает готовый файл.
2. **Безопасность**: Нет базы данных или PHP, которые можно взломать.
3. **Хостинг**: Бесплатно размещаются на GitHub Pages, GitLab Pages, Netlify или Vercel.
4. **Версионность**: Контент хранится как код (Markdown + шаблоны), легко отслеживать изменения через Git. 

Как они работают

Генератор берет исходные файлы (обычно в формате Markdown), обрабатывает их  шаблонами и превращает в набор HTML, CSS и JS-файлов. Они идеально  подходят для проектов, где контент меняется не каждую секунду (блоги,  документация, сайты-визитки).
