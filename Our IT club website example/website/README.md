## Technical Assignment for Development of the GITcube Educational Platform Website

### 1. General Provisions

**Project Name:** GITcube Educational Platform Website

**Project Objective:** Development of a modern, responsive website for an educational platform focused on programming education and exam preparation.

- **Main Objective:** Providing information about courses and services of the educational platform
- **System Objectives:**
  - Informing visitors about available courses
  - Demonstrating learning advantages
  - Collecting training applications
  - Ensuring website navigation

**Target Audience:**

- School students
- University students
- Adults interested in learning programming
- Applicants preparing for OGE and EGE exams

### 4. System Requirements

#### 4.1 Functional Requirements

- **Responsive design** for all devices
- **Navigation:**
  - Main menu
  - Mobile menu (hamburger)
  - Anchor menu
- **Main Sections:**
  - Homepage
  - Courses
  - About us
  - Contacts
- **Feedback form**
- **Course cards** with descriptions and detailed information links

#### 4.2 Non-functional Requirements

- **Performance:**
  - Page loading time not exceeding 3 seconds
  - Search engine optimization
- **Security:**
  - Protection against basic types of attacks
  - Input data validation
- **Accessibility:**
  - Compliance with WCAG 2.0 standards
  - Responsiveness across various devices

### 5. Composition and Content of Work

#### 5.1 Development Stages

- **Requirements analysis** and design
- **Markup** of HTML/CSS
- **Programming** of interactive elements
- **Testing** and debugging
- **Deployment** and launch

#### 5.2 Technical Tools

  - HTML5
  - CSS3
  - JavaScript
- **Backend:**
  - Form processing
  - Database interaction
- **Tools:**
  - Git for version control
  - Webpack for bundling

### 6. Documentation Requirements

- **Technical documentation** on website structure
- **User manual**
- **API documentation** (if required)

- **Preparatory stage:** 2 weeks
- **Development:** 4 weeks

## Образовательная платформа

**Образовательная платформа** — это цифровое пространство (LMS/VLE) для онлайн-обучения, которое  предлагает такие функции, как управление курсами, доставка контента,  оценка знаний (тесты, проекты) с автоматической проверкой и обратной  связью, отслеживание прогресса, а иногда и технические задания (как на  LeetCode или Codecademy) для развития практических навыков. Примеры  таких платформ: Coursera, edX, Canvas, Blackboard и специализированные  сайты для программирования. Эти платформы помогают преподавателям  назначать задания, а студентам — выполнять интерактивные практические  задания, часто с выдачей сертификатов.

### Основные характеристики и типы платформ

- **Общее образование и профессиональные навыки.** Такие платформы, как Coursera, edX, Udemy, LinkedIn Learning и  Skillshare, предлагают разнообразные курсы с заданиями, проектами и  сертификатами, некоторые из которых аккредитованы.
- **Школьное и академическое образование.** Google Classroom, Canvas и Blackboard Learn предоставляют мощные  инструменты для заданий, оценивания и взаимодействия со студентами в  учебных заведениях.
- **Программирование и технологии.** Codecademy, LeetCode и Pluralsight специализируются на интерактивных  задачах по программированию, практических упражнениях и сертификациях по навыкам.
- **Конструкторы платформ.** Thinkific, iSpring и BuddyBoss позволяют создателям разрабатывать собственные образовательные сайты с функциями для заданий.

### Как работают технические задания

- **Отправка и оценка.** Студенты загружают файлы (код, документы), отправляют их через  встроенные инструменты (например, IDE на Codecademy) или выполняют  интерактивные задания с автоматической оценкой или проверкой  преподавателем.
- **Типы контента.** Видеоуроки, интерактивные упражнения по программированию, тесты с  автоматической проверкой, проекты по программированию с тестами и  письменные отчёты.
- **Обратная связь.** Автоматическое обнаружение ошибок в коде, взаимная проверка работ, комментарии преподавателей и аналитика успеваемости.

### Примеры использования

- **Codecademy.** Студенты пишут код прямо в браузере и получают мгновенную обратную связь по заданиям.
- **Coursera/edX.** Предлагают сложные задания по программированию и проекты с оценкой коллегами для глубокого изучения материала.
- **Canvas/Blackboard.** Преподаватели создают папки для заданий, куда студенты загружают файлы с кодом, или предоставляют ссылки на внешние платформы, такие как  GitHub/LeetCode.
